## The full flow

```bash
jack new my-api --template api        # Create and deploy
jack services db create               # Add a database
jack db execute --write "CREATE TABLE items (id INTEGER PRIMARY KEY, name TEXT, created_at TEXT DEFAULT CURRENT_TIMESTAMP)"
jack ship                             # Push changes live
```

Your API is running. Database is wired. Took about 30 seconds.

## What you get

| Capability | How |
|-----------|-----|
| **Deploy** | `jack ship` — builds and pushes to production |
| **Database** | `jack services db create` — provisions D1, wires bindings |
| **Query** | `jack db execute "SELECT * FROM users"` — runs against production |
| **Secrets** | `jack secrets set API_KEY` — stored securely, available as `env.API_KEY` |
| **Logs** | `jack logs` — real-time production logs |
| **Storage** | `jack services storage create` — R2 bucket for file uploads |
| **Vector search** | `jack services vectorize create` — embeddings index |
| **Cron** | `jack services cron create "0 * * * *"` — scheduled tasks |
| **Custom domains** | `jack domain connect app.example.com` |

## Built for Claude Code

jack is the deployment platform for Claude Code. Every project includes context files, and there's a built-in MCP server so agents can deploy and manage services directly.

```bash
jack mcp serve                        # Start MCP server
```

Or install the [OpenClaw skill](https://clawhub.ai/hellno/jack-cloud) to teach your agent the full workflow:

```bash
npx clawhub@latest install jack-cloud
```

Key commands support `--json` for machine-readable output:

```bash
jack ship --json
jack db execute --json "SELECT count(*) as n FROM users"
```

## No lock-in

Everything jack creates is standard tooling — `wrangler.jsonc`, `package.json`, TypeScript. Your projects work without jack installed. Eject anytime with `wrangler deploy`.
